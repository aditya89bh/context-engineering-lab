# Project summary

A one-page overview of what `context-engineering-lab` contains, by the numbers and
by phase. For the full narrative see the [README](../README.md); for where files
live see [repository-layout.md](repository-layout.md) and
[repository-tour.md](repository-tour.md).

## What it is

A deterministic, dependency-free research workbench for the **context-engineering
problem**: how a bounded system decides what to retain, compress, retrieve,
prioritize, and forget. The experiments are the product. Every benchmark is
generated from a seed, measured against an oracle ceiling and content-blind
baselines, and regenerates bit-for-bit.

## Phase overview

| Phase | Theme | Adds |
| --- | --- | --- |
| 0–1 | Research design + core harness | items, budgets, runner, registry, seeding |
| 2 | Selection under budget | selection benchmark family + strategies |
| 3 | Compression | extractive compressors + fact-preservation benchmark |
| 4 | Temporal relevance | temporal strategies + temporal benchmarks |
| 5 | Retention / forgetting | retention policies + retention benchmarks |
| 6 | Attention allocation | allocators + attention benchmarks |
| 7 | Interaction effects | compositions + interaction benchmarks |
| 8 | Naturalistic context | email/meeting/support/revision/memory families |
| 9 | Cross-benchmark synthesis | profiles, dominance, oracle gap, failure, stability |
| 10 | Robustness | perturbations, robustness metrics, stress experiments |
| 11 | Release hardening | audit, docs, reproducibility, publication artifacts |

## Key outputs

- **Reports** (regenerated by the CLI): per-phase Markdown summaries
  (`summary.md`), the cross-benchmark `synthesis.md` (Phase 9), and the
  `robustness.md` report (Phase 10).
- **Publication documents**: [RESULTS.md](../RESULTS.md) (evidence),
  [FINDINGS.md](../FINDINGS.md) (interpretation),
  [LIMITATIONS.md](../LIMITATIONS.md), [FUTURE_WORK.md](../FUTURE_WORK.md).
- **Reproducibility**: [reproducibility.md](reproducibility.md) and a checked-in
  example artifact under [examples/](examples/), validated by the test suite.
- **Headline readings** (benchmark-specific; see RESULTS/FINDINGS): value-
  estimating strategies such as `retention->selection` and `salience-retention`
  sit closest to the oracle across benchmarks, while the same strategies are the
  most exposed when the signal they trust is perturbed.

## Running it

```bash
pip install -e ".[dev]"
context-lab list
context-lab run-phase9 --output artifacts/phase9   # synthesis of all suites
```
